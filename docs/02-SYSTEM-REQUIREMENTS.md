# Системийн шаардлагын тодорхойлолт (System Requirements Specification)

## 1. Төслийн танилцуулга

### 1.1 Зорилго
Microservices архитектур ашиглан уулзалт, эвент төлөвлөлтийн систем (EventMN) хөгжүүлэх. Энэхүү систем нь хэрэглэгчдэд эвентүүдийг хайх, захиалга хийх, зохион байгуулагчдад эвент үүсгэх, удирдах боломжийг олгоно.

### 1.2 Хамрах хүрээ
- Хэрэглэгчийн бүртгэл, нэвтрэлт, баталгаажуулалт
- Эвент үүсгэх, засварлах, устгах
- Суудал сонгох, түгжих механизм
- Захиалга үүсгэх, баталгаажуулах, цуцлах
- Мэдэгдэл илгээх (email, in-app)
- Админ удирдлага

### 1.3 Хэрэглэгчийн төрлүүд

| Төрөл | Тодорхойлолт |
|-------|--------------|
| Зочин (Guest) | Бүртгэлгүй хэрэглэгч - зөвхөн эвент үзэх |
| Хэрэглэгч (User) | Бүртгэлтэй хэрэглэгч - захиалга хийх |
| Зохион байгуулагч (Organizer) | Эвент үүсгэх, удирдах |
| Админ (Admin) | Бүх системийг удирдах |

### 5.2 Технологийн стек

| Давхарга | Технологи |
|----------|-----------|
| Frontend | Next.js 14, React, Tailwind CSS, Zustand |
| Gateway | Next.js API Routes |
| Services | Next.js API Routes, Node.js 20 |
| Database | PostgreSQL 15 |
| Cache | Redis 7 |
| Message Broker | RabbitMQ 3 |
| Container | Docker, Docker Compose |
| ORM | Prisma (PostgreSQL) |

### 5.3 API Standards

- RESTful API design
- JSON format for request/response
- HTTP status codes: 200, 201, 400, 401, 403, 404, 500
- Pagination: `?page=1&limit=10`
- Filtering: `?category=CONCERT&status=PUBLISHED`
- Sorting: `?sort=createdAt&order=desc`

### 5.4 Authentication Flow

```
1. Register -> Send OTP -> Verify OTP -> Account Active
2. Login -> Generate Access Token (15min) + Refresh Token (7d)
3. Access API -> Validate Access Token -> Allow/Deny
4. Token Expired -> Use Refresh Token -> New Access Token
5. Refresh Token Expired -> Re-login
```
